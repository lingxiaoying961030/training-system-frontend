🚫 任务9：不通过标记 — 三种方案对比

场景：学员在测验中用完所有机会仍未通过（status=failed, attempts exhausted），当前在学员进度/培训地图中仍显示"进行中"，无法区分。
下面展示三种方案，每种包含：培训地图节点 + 学员进度表格 两个视角。
🅰️ 方案A：在原有状态上增加"不通过"（推荐）
当学员测验次数用完且未通过时，关卡状态从 active 变为 failed。培训地图显示红色标记，学员列表显示❌标签。
管理员重置测验后，状态恢复为 active。

📊 三种方案对比

维度🅰️ failed 状态🅱️ 子标签🅲 blocked 状态
改动量中等（状态流转+UI）最小（仅UI）较大（新状态+全链路）
语义清晰度⭐⭐⭐ 最清晰⭐⭐ 有矛盾⭐⭐⭐ 清晰
和现有体系兼容✅ stage_progress 已有 failed✅ 不改状态⚠️ 需全链路加新值
管理员感知❌红色一眼看到⚠️橙色，需看两个标签⚠️橙色，但只有一个
学员感受较直接（不通过）温和（还在进行中）温和（需要帮助）
推荐指数⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐
我的推荐：方案A。原因：
1. training_stage_progress 表本身已有 failed 状态值，培训地图也已经写好了 failed 的图标/颜色/文案，只是当前后端没有在"测验耗尽"时设置它
2. 只需在后端提交测验的逻辑里加一个判断：如果 score < passScore 且 attemptCount >= maxAttempts → 设 stage_progress = failed
3. 管理员重置测验次数时，同步把 stage_progress 恢复为 active
4. 改动量小，语义最清晰
